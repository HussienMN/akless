1. Motivation, Learning Goals und Requirements

The up coming modular web is a big game changer. Instead of building monolithic web applications, you compose web apps together from already existing web components, like the Lego way. Each web component that is published under a free software license increases the functionality of the hole web, not only of a specific platform. The browser is the platform. So if you build a component-based web app, you build it for 3 billion peoples. Such kind of web apps are embeddable in any web-based content and can be used in form of a simple HTML tag. There are no restrictions in complexity and application domain. Here ccm comes in. It allows you to develop such kind of web components. This learning unit gives you a quick overview what ccm is and what it is used for. Then we see how you can work with ccm-based web components without the need of any programming knowledge. Never the less it's useful to already know the markup language HTML(5) and, if you know JavaScript, that you have already realized that JavaScript has dramatically improved and cross-browser incompatibility is no more a criterion for exclusion.

1.1 Learning Goals

  • I know what ccm is and what it is used for.
  • I know how I can work with ccm-based web components.

1.2 Requirements

  • It's useful but not mandatory to already know the markup language HTML(5).
  • I know that JavaScript has improved and cross-browser incompatibility is no more a criterion for exclusion.

2. Content:

2.1 Client-side Component Model (ccm)

ccm stands for Client-side Component Model and is a model for composing, embedding and running web components inside the browser. Our implementation consists of a single JavaScript framework ccm and many ccm-based web components.

2.2 Web Component Cloud (W2C)

Here you see our already developed and ready to use ccm-based web components. We are currently working on places where you can find, publish and rate components and where you can create web apps out of them. These places are the visual interface of that, what we call a Web Component Cloud (W2C).

2.10 Packaging Content as App

Let's clarify some terms. In the field of digital learning we distinguish between digital learning content, applications and platforms. In case of ccm the platform is the browser. The applications are made up of the ccm-based web components. But what about the content? We have a content component which allows you to package any web-based content as a ccm-based app. For example: This video is a part of an Open Education Resource (OER).

Because the content of this learning unit is packaged as ccm-based app, it now can be reused in any web-based content like already explained before.

2.11 icOER and ECL

Notice that you are able to build interactive and collaborative OERs (icOER) with ccm-based web components. People can use these icOERs from different access points. Together with the aspect of the cross-platform collaboration this leads to what we call Embedded Collaborative Learning (ECL). See our paper to see the definition of ECL.
