1. Motivation, Learning Goals und Requirements

The up coming modular web is a big game changer. Our own implementation of the modular web is our own framework Client-side Component Model, shortly ccm. With ccm, you can compose web apps from already existing web components, just like Lego brick building. Each web component that is published under a free software license increases the functionality of the web as a whole, not a single platform only. In other words: The browser is the platform and web components extend the powerful capabilities of the browser. So if you build web components, you build it for potentially 3 billion people. Such kind of web components are embeddable in any web-based content and can simply be used via a custom HTML tag. There are no restrictions in complexity and application domain. ccm allows to develop such kind of web components. This learning unit gives you a quick overview what ccm is about and what it is used for. Then we see how you can work with ccm-based web components without programming skills.

1.1 Learning Goals

  • I know what ccm is and what it is used for.
  • I know how I can work with ccm-based web components.

1.2 Requirements

  • It's useful but not mandatory to already know the markup language HTML and its modern successor HTML5.
  • It is required to know that JavaScript has improved and cross-browser incompatibility is no more a reason for exclusion.

Each ccm-based app can also be deployed on any mobile devices as mobile web app. This has the advantage that people that use our course app could access it inside the learning platform, but also as stand-alone mobile web app. Remember: The collaboration works between all instances, no matter from which platform it is accessed.

2.10 Packaging Content as App

Let's clarify some terms. In the field of digital learning we distinguish between digital learning content, applications and platforms. In case of ccm the platform is the browser. The applications are made up of ccm-based web components. But what about the content? We have a content component which allows for packaging any web-based content into a ccm-based app. For example: This video is a part of an Open Education Resource (OER), packaging both, contents and software as well.

Because the content of this learning unit is packaged as ccm-based app, it can now be reused in any web-based content.

2.11 icOER and ECL

Notice that interactive and collaborative OERs (icOER) can be built with ccm-based web components. These icOERs can be used from any access points. Together with cross-platform collaboration this leads to Embedded Collaborative Learning (ECL), as defined in our paper1.
